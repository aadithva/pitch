# DesignShift — explainer (layer by layer)

How this deck + video were generated by the `deck-to-video` skill, documented at
every layer.

---

## Layer 0 — The source project
- **DesignShift** — a Microsoft FHL "AI-led design lifecycle" hub, featuring
  **HatchLab** (an AI idea-graph / brainstorming portal).
- **Stack:** static HTML site (served directly); HatchLab is `jamlab/index.html`.
  Hosted on VibeHub (`vibehub.microsoft.com/app/designshift`).
- Source: `~/Documents/Claude workspace/projects/Website MSAI`.

## Layer 1 — Visual style (auto-extracted) → `visual-style/`
- **Background** `#0e0e0e`, **primary** `#ffffff`, **accent** `#faab0a` (amber)
- **Heading** Unbounded · **body** DM Sans · **mono** Space Mono
- Source signals: the site's **Google Fonts `<link>`** (Unbounded/DM Sans/Space
  Mono, exact weights), brand tokens, and logo palette. Accent auto-picked as the
  most-saturated brand color.
```bash
node scripts/extract_style.mjs "<project>/Website MSAI" --out build/msai
```

## Layer 2 — The deck → `deck/`
10 slides; the **Solution / AI-lab / HatchLab / Resources** slides are `showcase`
layouts that frame the **real captured UI** (Layer 5) — small text left, enlarged
screenshot right.

| # | Slide | Notes |
|---|-------|-------|
| 1 | Cover | "DesignShift" in Unbounded + mono eyebrow |
| 2 | Problem | AI help is scattered across tools |
| 3 | Users | Designers & makers racing through Fix · Hack · Learn |
| 4 | Insight | One AI-led design lifecycle, in one place |
| 5 | Solution | **showcase** — DesignShift home (Start with a task) |
| 6 | How / AI lab | **showcase** — AI lab screen |
| 7 | **HatchLab** | **showcase** — "Hatch your idea" canvas (the flagship) |
| 8 | Resources | **showcase** — grab-and-go prompts |
| 9 | What's next | roadmap |
| 10 | CTA | "Launch DesignShift. Hatch your next idea." |
```bash
node scripts/render_deck.mjs build/msai/slides.json build/msai/theme.json --out build/msai/deck.html
node scripts/export_pdf.mjs build/msai/deck.html --out build/msai/deck.pdf --png build/msai/frames
```

## Layer 3 — Narration / audio
Per-slide `narration`, synthesized with **edge-tts** (`en-US-AriaNeural`). Total ≈ 60s.
```bash
./.venv/bin/python scripts/tts.py build/msai/slides.json --out build/msai
```

## Layer 4 — Final video → `final-video/`
`render_video.mjs` → clean narrated MP4 (no captions/highlight).
```bash
node scripts/render_video.mjs build/msai/deck.html build/msai --out build/msai/final.mp4 --fps 12
```
*(A variant with the live recording spliced in as a "live demo" section can be
produced with `splice_demo.mjs` — kept separate here per the demo/final split.)*

## Layer 5 — Live UI demo → `demo-video/`
`capture_demo.mjs` served the static site and captured the real UI + a flow recording.
- Routes captured: `/` (home), `/ai-lab.html`, `/resources.html`, `/jamlab/index.html` (**HatchLab**)
```bash
node scripts/capture_demo.mjs "<project>/Website MSAI" --out build/msai \
  --routes "/,/ai-lab.html,/resources.html,/jamlab/index.html" --record true
```
