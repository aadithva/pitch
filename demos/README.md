# Demos — `deck-to-video` skill

Explainer **decks + narrated videos** generated by the [`deck-to-video`](../deck-to-video/)
skill, run on real vibe-coded projects. Each project folder is organized by layer.

## Projects
| Project | What it is | Folder |
|---|---|---|
| **DesignShift** | Microsoft FHL — AI-led design lifecycle (incl. **HatchLab**, an AI idea graph) | [`designshift/`](./designshift/) |
| **FlowSense** | AI interaction-quality scoring for UX task-flow recordings (Next.js + Azure) | [`flowsense/`](./flowsense/) |

## Folder layout (per project)
```
<project>/
├── README.md         # layer-by-layer documentation of how it was made
├── visual-style/     # LAYER 1 — the brand auto-extracted from the project
│   ├── theme.json    #   canonical tokens (colors, fonts, weights)
│   ├── theme.css     #   compiled reveal.js variables
│   └── style.md      #   human-readable palette + fonts summary
├── deck/             # LAYER 2 — the standalone deck
│   ├── *-deck.pdf
│   ├── *-deck.html
│   └── slides.json   #   the deck spec the deck was rendered from
├── final-video/      # LAYER 4 — the narrated explainer video (clean: no captions/highlight)
│   └── *-explainer.mp4
└── demo-video/       # LAYER 5 — the live product UI, captured by running the app
    ├── *-ui-recording.mp4
    └── screenshots/
```

## The layers (pipeline)
1. **Visual style** — `extract_style.mjs` reads the project's design tokens / CSS
   vars / fonts / logo and produces a brand theme. → `visual-style/`
2. **Deck** — an LLM writes `slides.json` (the deck spec); `render_deck.mjs`
   renders a self-contained, brand-themed reveal.js deck. → `deck/`
3. **Narration / audio** — per-slide scripts → `tts.py` (edge-tts) → voice + word
   timings. *(audio lives in the skill's build dir; documented per project.)*
4. **Final video** — `render_video.mjs` plays the deck headlessly, captures it
   frame-by-frame, and muxes the narration → MP4. → `final-video/`
5. **Live demo** — `capture_demo.mjs` runs the actual project UI and records
   per-route screenshots + a flow recording. → `demo-video/`

See each project's `README.md` for the exact commands and slide-by-slide details.

> These are local build artifacts (videos/PDFs) and are git-ignored.
