# FlowSense — explainer (layer by layer)

How this deck + video were generated by the `deck-to-video` skill, documented at
every layer.

---

## Layer 0 — The source project
- **FlowSense** — "analyzes UX task-flow recordings and scores interaction quality
  using a 7-category rubric."
- **Stack:** turborepo monorepo; **Next.js + shadcn/ui + Tailwind** web app
  (`frontend/`), Azure-native (Entra ID auth, Azure SQL, Blob, **Azure OpenAI**).
- Source: `~/Documents/Claude workspace/projects/flowsense`.

## Layer 1 — Visual style (auto-extracted) → `visual-style/`
`extract_style.mjs` read the project and produced an authentic theme:
- **Background** `#000000`, **surface** `#0e0e10`, **text** `#fafafa`
- **Primary / accent** `#07bdd5` (cyan) — the brand color
- **Heading + body** Space Grotesk, **mono** Space Mono
- Source signals: **shadcn HSL color variables** (e.g. `--primary: 187 94% 43%`)
  and **`next/font/google`** imports. (Both were added to the extractor for this
  project — they're a very common stack.)
```bash
node scripts/extract_style.mjs <project>/frontend --out build/flowsense
```

## Layer 2 — The deck → `deck/`
An LLM wrote `slides.json` (10 slides) following the project-explainer blueprint;
`render_deck.mjs` rendered a self-contained, brand-themed reveal.js deck.

| # | Slide | Notes |
|---|-------|-------|
| 1 | Cover | "FlowSense" + mono eyebrow |
| 2 | Problem | Judging interaction quality is slow & subjective |
| 3 | Users | UX researchers, designers, PMs |
| 4 | Insight | Interaction quality is visible frame-by-frame |
| 5 | **Solution** | **showcase of the real upload UI** (captured in Layer 5) |
| 6 | How it works | native **flow** diagram: Recording → Keyframes → Azure OpenAI → rubric → score |
| 7 | The rubric | the real **7-category** rubric (pulled from the codebase) |
| 8 | Azure-native | Entra auth · managed identity · signed webhooks |
| 9 | What's next | per-frame evidence · self-consistency · Cohen's kappa |
| 10 | CTA | "Stop guessing. Score the flow." |
```bash
node scripts/render_deck.mjs build/flowsense/slides.json build/flowsense/theme.json --out build/flowsense/deck.html
node scripts/export_pdf.mjs build/flowsense/deck.html --out build/flowsense/deck.pdf --png build/flowsense/frames
```

## Layer 3 — Narration / audio
Each slide carries a `narration` script (complements the slide, doesn't read it).
Synthesized with **edge-tts** (`en-US-AriaNeural`), which also yields word timings.
Total ≈ 77s.
```bash
./.venv/bin/python scripts/tts.py build/flowsense/slides.json --out build/flowsense
```

## Layer 4 — Final video → `final-video/`
`render_video.mjs` plays the deck headlessly, captures it frame-by-frame, and muxes
the narration. **Clean by default** — no caption bar, no on-slide highlight.
```bash
node scripts/render_video.mjs build/flowsense/deck.html build/flowsense --out build/flowsense/final.mp4 --fps 12
```

## Layer 5 — Live UI demo → `demo-video/`
`capture_demo.mjs` **ran the actual Next.js app** (`next dev`), auto-discovered
routes, and captured screenshots + a flow recording.
- Routes captured: `/` (Analyze UX Flows — upload), `/history`, `/settings`
- The upload screen is used as the Solution slide's showcase (Layer 2).
```bash
node scripts/capture_demo.mjs <project>/frontend --out build/flowsense --max-routes 5 --record true
```
> Note: FlowSense is Azure/Entra-gated, but its **Next.js dev server boots with the
> repo's `.env.local`** and renders the real UI, so the app **was** capturable.
> (For a fully auth-walled instance, point `capture_demo --url` at a deployed URL.)
