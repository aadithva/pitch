Use the **pitch** skill in **SCRIPT-ONLY** mode for the target project
(`$ARGUMENTS` if given, else the current working directory).

1. **Understand** the project (README, manifest, key source).
2. Following `references/deck-blueprint.md` (the 10–12 slide arc) and
   `references/narration-rules.md` (pacing, "complement don't repeat",
   conversational tone), write a **per-slide narration script** — the spoken
   talk-track for an explainer.

Output as markdown: for each slide, the slide title/idea and its spoken script.
Do **not** render a deck, synthesize audio, or produce video.
